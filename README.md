# Satellite Image Classification

A machine learning pipeline that classifies satellite images into four land-use categories using the EuroSAT dataset.

**Classes:** Agriculture · Vegetation · Urban · Water  
**Model:** Random Forest (300 trees)  
**Stack:** Python · scikit-learn · FastAPI

## Setup

### Requirements

- Python 3.11+
- pip

### 1. Clone and create virtual environment

```bash
git clone https://github.com/your-username/satellite-image-classification.git
cd satellite-image-classification

python -m venv .venv
source .venv/bin/activate        # Linux / macOS
.venv\Scripts\activate           # Windows
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Add the dataset

Download [EuroSAT](https://github.com/phelber/EuroSAT) and place the files under `data/`:

```
data/
├── train.csv
├── validation.csv
├── test.csv
└── <image folders>/
```

### 4. Run the pipeline

Each command must run in order — each step produces a file the next one consumes.

```bash
# Step 1 — remap classes and balance training data
python src/preprocessing.py

# Step 2 — extract features and train the model
python src/train.py

# Step 3 — start the API
uvicorn src.api:app --reload
```

### 5. Test the API

Open the auto-generated Swagger docs:

```
http://localhost:8000/docs
```

Or send a request directly:

```bash
curl -X POST http://localhost:8000/predict \
  -F "file=@your_image.jpg"
```

**Response:**
```json
{
  "filename": "your_image.jpg",
  "label": "Agriculture",
  "confidence": 0.8731,
  "probabilities": {
    "Agriculture": 0.8731,
    "Vegetation": 0.0842,
    "Urban": 0.0291,
    "Water": 0.0136
  }
}
```

## Architecture

### Pipeline flow

```
preprocessing.py  →  outputs/preprocessing/*.csv
        ↓
train.py          →  outputs/model/model.joblib
        ↓
predict.py        →  loads model.joblib (lazy, on first call)
        ↓
api.py            →  POST /predict  →  JSON response
```

Each file has a single responsibility. They communicate through files on disk — no file imports the previous one directly.

---

### `preprocessing.py`

Prepares the data before training. Solves two problems: class granularity and class imbalance.

**`remap_splits()`**  
Maps the 10 original EuroSAT classes to 4 superclasses using a dictionary. Classes with no mapping (River, Highway) are saved to `{split}_to_infer.csv` rather than discarded. Adds a numeric `Label` column (0–3) because scikit-learn requires integer targets.

| Original classes | Superclass |
|---|---|
| AnnualCrop, PermanentCrop, Pasture | Agriculture |
| Forest, HerbaceousVegetation | Vegetation |
| Industrial, Residential | Urban |
| SeaLake | Water |

**`augment_train()`**  
After remapping, classes that group fewer original classes end up with fewer training samples. A model trained on imbalanced data will favour the majority class regardless of its actual performance on minority classes.

The solution is rotation augmentation — applied **to the training split only**. Validation and test are never touched so that metrics reflect the real-world distribution.

For each minority class, images are rotated by 90°, 180°, and 270° and appended to `train_remapped.csv` until the class reaches the majority count or all 3 rotations are exhausted. A 4th rotation (360°) would reproduce the original image, so it is never generated.

Rotation is valid for satellite imagery because top-down images have no canonical orientation — a crop field rotated 90° is still a valid crop field.

---

### `train.py`

Six functions, each with a single responsibility.

**`check_data_leakage()`**  
Runs before any training. Reads the `ImagePath` column from each CSV and computes set intersections between train and the other two splits. If any image path appears in more than one split, the script raises a `RuntimeError` and stops completely — contaminated metrics are worse than no metrics.

**`color_histogram()`**  
For each of the 3 RGB channels, divides the intensity range 0–255 into 32 bins and counts the proportion of pixels in each bin. The output is a vector of 96 values that describes the colour distribution of the image. Water images concentrate in the blue channel; vegetation in green; urban areas spread evenly across all three channels because concrete and asphalt are grey.

```
Output: 32 bins × 3 channels = 96 values
```

**`hog_features()`**  
Captures edge directions and texture. The image is divided into 8×8 pixel cells. Within each cell, gradients measure how brightness changes between neighbouring pixels. Those gradients are binned into 8 directions (0°, 22°, 45°… 157°) and normalised across 2×2 cell blocks to remove sensitivity to lighting variation.

Agriculture shows dominant orientations — crop rows and field boundaries create regular, directional edges. Forest shows near-uniform gradient directions — organic texture has no dominant angle. Urban areas show strong vertical and horizontal edges from buildings and roads.

```
HOG_PIXELS = 8   →  8×8 px cells  →  64×64 image = 8×8 grid of cells
HOG_CELLS  = 2   →  normalise across 2×2 cell blocks
```

**`build_model()`**  
Returns a scikit-learn `Pipeline` with two steps:

1. `StandardScaler` — normalises all features to mean=0, std=1. Required because color histogram and HOG values live on different numerical scales.
2. `RandomForestClassifier` — ensemble of 300 decision trees. Each tree trains on a random data and feature subset; the final prediction is a majority vote.

| Parameter | Value | Reason |
|---|---|---|
| `n_estimators` | 300 | More trees → more stable predictions. Below ~100 accuracy drops noticeably; above 300 the gain is marginal. |
| `class_weight` | balanced | Second layer of imbalance protection after augmentation. Assigns higher training loss to underrepresented classes. |
| `random_state` | 42 | Fixed seed — running the script twice produces the identical model. |
| `n_jobs` | -1 | Parallelises tree construction across all CPU cores. |

**`evaluate()`**  
Called twice: validation during development, test once at the end for the final reported number.

Reports two metrics: `accuracy` (fraction correct — easy to game on imbalanced data) and `MCC` (Matthews Correlation Coefficient — robust across all four classes, ranges from -1 to +1, near zero for a majority-class baseline regardless of accuracy).

**`save_outputs()`**  
Writes three files to `outputs/model/`:

- `model.joblib` — serialised trained model, loaded by `predict.py`
- `metrics.json` — accuracy, MCC, F1 per class, confusion matrix
- `feature_importance.json` — top 20 features the forest used most, providing partial interpretability for an otherwise black-box model

---

### `predict.py`

Loads the model and runs inference. One public function: `predict()`.

**Lazy loading**  
`_model` starts as `None`. The model is only loaded from disk on the first call to `predict()`. Loading at import time would crash the API on startup if `model.joblib` does not exist yet. After the first load, the model is cached in memory — subsequent calls skip the disk read entirely.

**`predict(image)`**  
Applies the same feature extraction pipeline as training (resize → color histogram → HOG), runs `model.predict_proba()` to get the full probability distribution across all four classes, and returns a typed `PredictionResult` dataclass:

```python
@dataclass
class PredictionResult:
    label:         str            # predicted class name
    confidence:    float          # probability of the predicted class
    probabilities: dict[str, float]  # full distribution across all 4 classes
```

---

### `api.py`

A thin FastAPI wrapper around `predict.py`. Single endpoint.

**`POST /predict`**  
Accepts a multipart image upload, validates it, calls `predict()`, and returns JSON.

| Status | Meaning |
|---|---|
| 200 | Prediction successful |
| 400 | Uploaded file is not a valid image (client error) |
| 503 | `model.joblib` missing — `train.py` has not been run yet |

400 and 503 are intentionally different codes — they represent different failure modes and should not both return 500.

FastAPI auto-generates interactive documentation at `/docs`.
